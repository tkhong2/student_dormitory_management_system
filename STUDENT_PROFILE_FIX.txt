========================================
VẤN ĐỀ VÀ CÁCH KHẮC PHỤC - TRANG HỒ SƠ SINH VIÊN
========================================

VẤN ĐỀ:
--------
Khi sinh viên đăng nhập và truy cập trang "Hồ sơ cá nhân", gặp lỗi 404:
"Không tìm thấy hồ sơ sinh viên"

NGUYÊN NHÂN:
-----------
- Khi user đăng nhập, thông tin tài khoản được lưu trong database BillingMaintenanceService (Users table)
- Thông tin sinh viên được lưu trong database ContractStudentService (Students table)  
- Để trang hồ sơ hoạt động, cần có liên kết: Students.UserId = Users.Id
- Lỗi xảy ra khi không tồn tại bản ghi Student có UserId tương ứng với tài khoản đăng nhập

VÍ DỤ:
------
- User đăng nhập với username "student01" có Id=17 (trong BillingMaintenanceService)
- Backend tìm kiếm: GET /api/students/by-user/17
- Nếu không có bản ghi Student nào có UserId=17 → Trả về 404

CÁCH KHẮC PHỤC:
===============

BƯỚC 1: XÁC ĐỊNH USER ID
-------------------------
Mở Developer Tools trong trình duyệt (F12), vào Console và chạy:
```javascript
const user = JSON.parse(localStorage.getItem('user'));
console.log('User ID:', user.id);
console.log('Username:', user.username);
console.log('Full Name:', user.fullName);
```

BƯỚC 2: TÌM STUDENT RECORD PHÙ HỢP
----------------------------------
Chạy script PowerShell để kiểm tra:
```powershell
cd backend
./diagnose-student-profile.ps1
```
Hoặc kiểm tra trực tiếp trong database:
```sql
-- Trong database ContractStudent
SELECT Id, StudentCode, FullName, Email, UserId
FROM Students
WHERE Email = 'email_cua_ban@example.com';
```

BƯỚC 3: LIÊN KẾT STUDENT VỚI USER
----------------------------------
Có 3 phương án:

PHƯƠNG ÁN A: Cập nhật UserId cho Student đã tồn tại (KHUYẾN NGHỊ)
```sql
-- Trong database ContractStudent  
-- Thay <student_id> bằng Id từ bước 2
-- Thay <user_id> bằng Id từ bước 1
UPDATE Students
SET UserId = <user_id>
WHERE Id = <student_id>;

-- Ví dụ: UPDATE Students SET UserId = 17 WHERE Id = 3;
```

PHƯƠNG ÁN B: Tạo Student mới và liên kết với User
```sql
-- Trong database ContractStudent
INSERT INTO Students (
    StudentCode, Faculty, Major, AcademicYear, ClassCode,
    FullName, Gender, DateOfBirth, Nationality,
    Phone, Email, UserId, IsActive, ProfileCompletionPct,
    PermanentAddress, IdentityCard, IdentityCardIssuedDate, IdentityCardIssuedPlace
) VALUES (
    'SV001234',                          -- Mã sinh viên
    'Công nghệ thông tin',               -- Khoa
    'Công nghệ thông tin',               -- Ngành
    1,                                   -- Năm học
    'K17-CNTT',                          -- Lớp
    'Nguyễn Văn A',                      -- Họ tên
    'Male',                              -- Giới tính
    '2000-01-01',                        -- Ngày sinh
    'Việt Nam',                          -- Quốc tịch
    '0987654321',                        -- SĐT
    'email@example.com',                 -- Email
    17,                                  -- UserId (từ bước 1)
    1,                                   -- IsActive
    50,                                  -- ProfileCompletionPct
    'Địa chỉ thường trú',               -- Địa chỉ
    '001234567890',                      -- CCCD
    '2018-01-01',                        -- Ngày cấp
    'Hà Nội'                            -- Nơi cấp
);
```

PHƯƠNG ÁN C: Sử dụng Admin Panel
1. Đăng nhập với tài khoản Admin
2. Vào menu "Quản lý sinh viên"
3. Chỉnh sửa hồ sơ sinh viên cần liên kết
4. Chọn User tương ứng từ dropdown

BƯỚC 4: KIỂM TRA
----------------
1. Đăng xuất và đăng nhập lại
2. Truy cập trang "Hồ sơ cá nhân"
3. Kiểm tra xem dữ liệu đã hiển thị đúng chưa

Hoặc kiểm tra bằng API:
```powershell
# Thay <user_id> bằng User ID của bạn
curl http://localhost:5000/api/students/by-user/<user_id>
```

LƯU Ý:
------
- Mỗi User chỉ nên liên kết với 1 Student
- Mỗi Student chỉ nên liên kết với 1 User
- Email trong User và Student nên giống nhau để dễ quản lý
- Sau khi sửa database, nên khởi động lại backend services
